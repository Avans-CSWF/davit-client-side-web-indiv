Functionalities

Here are all the things that this application can do.

User Registration and Authentication

ID: FR-1

Requirement: The application must provide the option for user registration.


Description: Users should be able to create an account and log in. User should be able to edit their personal information.


Moscow priority: Must have

User Listing and Details

ID: FR-2

Requirement: The application must provide the functionality for users to view a list of all registered users and their details.


Description: Users should have access to a page where they can see a list of all registered users. They should be able to view details such as username, email address, and any other relevant information associated with each user.


Moscow priority: Must have

Sell Products

ID: FR-3

Requirement: Users should be able to create their own products, edit and delete them.


Description: Each user should be able to create their own products, so that other users can purchase.


Moscow priority: Must have

Product Display and Purchase

ID: FR-4

Requirement: Users should be able to view all the products on the sale and their details.


Description: Products should be displayed with relevant information and an image. Users should be able to click on the products and see the details of it.


Moscow priority: Must have

Shopping Cart Management

ID: FR-5

Requirement: The application must allow users to manage their shopping cart.


Description: Users should be able to view their shopping cart, add items to it and remove items from it. Additionally, users should be able to see the total price of their shopping basket.


Moscow priority: Must have

Favorite Products

ID: FR-6

Requirement: Users must be able to mark products as favorites.


Description: Users should have the option to add products to their list of favorites for easy access later. They should also be able to view their list of favorite products and remove products from the list. They should also be able to click on a product and see the details of it.


Moscow priority: Should have

Recommendations

ID: FR-7

Requirement: For a Product in the shopping cart, you can see which other products have been added to the shopping cart for purchase.


Description: For example, if a user buys a smartphone, you should be able to see that a phonecase is often also ordered with the phone.


Moscow priority: Should have

Multistore Functionality

ID: FR-8

Requirement: Users should be able to create, edit, and delete their own shops, as well as view all other shops created by different users. The ultimate goal is for users to sell products through their shops.


Description: The system currently supports the creation, updating, and deletion of shops, as well as reading and viewing detailed information about them. However, the ultimate objective is for users to sell products through their shops.


Moscow priority: Clould have
